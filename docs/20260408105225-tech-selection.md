# 技術選定

## 言語: Go

- CLI ツールとしてシングルバイナリ配布が可能
- チームの希望

## SQL パーサー: go-zetasql (github.com/goccy/go-zetasql)

Google 公式 ZetaSQL の Go バインディング。

### 選定理由

clearing_on_gcp リポジトリの全53 SQL ファイルで検証を実施。

| パーサー | 言語 | 成功率 | 備考 |
|---|---|---|---|
| **go-zetasql** | Go | **53/53 (100%)** | 全機能対応 |
| sqlglot | Python | 52/53 (98%) | BEGIN/END ブロック内のテーブル抽出不可 |
| sqlparser-rs | Rust | 46/53 (87%) | WHILE DO 構文未対応 |

go-zetasql のみが BigQuery スクリプティング構文（WHILE DO, BEGIN/END, DECLARE）を含む全ファイルをパースし、テーブル参照を抽出できた。

### 検証で確認した対応構文

- バッククォート付き fully-qualified テーブル参照
- WITH / WITH RECURSIVE (CTE)
- MERGE INTO
- CREATE OR REPLACE TABLE ... AS
- QUALIFY
- BEGIN/END スクリプティング（DECLARE, WHILE DO, SET, IF）
- UNNEST
- JSON 関数群（JSON_VALUE, JSON_QUERY_ARRAY 等）
- SAFE_CAST, SAFE_DIVIDE, SAFE.PARSE_DATE
- ウィンドウ関数（フレーム指定含む）
- ARRAY_AGG（IGNORE NULLS, ORDER BY, LIMIT 修飾子付き）
- TRUNCATE TABLE
- ALTER TABLE ADD COLUMN IF NOT EXISTS

### トレードオフ

- CGO 依存（ZetaSQL は C++ ライブラリ）
- 初回ビルド: 約5分、キャッシュ後: 約20秒
- cross-compile には OS 別ビルド環境が必要
- 対象は macOS + Linux のみで十分なため、CI で OS 別 runner を使えば対応可能
